# <a name="ado-glossary"></a>ADO 术语表

**适用于**：Access 2013、Office 2013

## <a name="a"></a>A

**绝对 URL**

一个完全限定的 URL, 用于指定驻留在 Internet 或 intranet 上的资源的位置。 另请参阅**url**和**相对 url**。

**ActiveX 控件**

在设计时或运行时通常具有 visual 元素的自注册进程内 COM 组件。 ActiveX 控件还能够与活动文档容器 (如 Microsoft Internet Explorer) 进行通信。

**ADISAPI (高级数据 Internet 服务器应用程序编程接口)**

提供分析、自动化控制、 **Recordset**封送和 MIME 打包的 ISAPI DLL。 ADISAPI 组件通过 Internet 信息服务 (IIS) 提供的 API 进行工作。 另请参阅**ISAPI**。

**聚合函数**

在查询中, 使用表的某一列中的所有行计算值的函数 (如 COUNT、AVG 或 STDEV)。 在编写表达式和编程中, 可以使用 SQL 聚合函数 (包括上面列出的三个) 和域聚合函数来确定各种统计信息。

**alias**

您为 SQL SELECT 语句中的列或表达式赋予的替代名称, 通常较短或更有意义。 例如, BobSales 是以下 SELECT 语句中的别名: "SELECT wr-Sales as BobSales from SalesDB"。 可以使用别名在**rds.datacontrol**对象上将列动态分配给控件绑定。

**单元线程**

一个 COM 线程模型, 其中对象的所有调用都发生在一个线程上。 在单元线程中, COM 将同步和封送调用。 另请参阅**COM**。

**异步操作**

在不等待操作完成的情况下, 将控制返回给调用程序的操作。 在操作完成之前, 将继续执行代码。 另请参阅**同步操作**。

**位掩码**

**bookmark**

唯一标识一组行中的行的标记, 以便用户可以快速导航到该数据行。

**业务对象**

执行定义的一组操作 (如数据验证或业务规则逻辑) 的对象。 业务对象通常驻留在中间层上。

**业务规则**

验证编辑、登录验证、数据库查找、策略和算法转换的组合构成了企业开展业务的方式。 也称为*业务逻辑*。

## <a name="c"></a>C

**计算表达式**

一个不是常量, 但其值取决于其他值的表达式。 计算的表达式必须从其他源 (通常在其他字段或行中) 获取和计算值, 才能对其进行计算。

**第二章**

对数据源中的行区域的引用。 在 ADO 中, 一章通常是对另一个**Recordset**的引用。

章节列使您可以定义*父子*关系，其中*父*是包含章节列的 **Recordset**，而*子*是章节所表示的 **Recordset**。

**chapter-alias**

引用追加到父项的列的别名。

**字符集**

一组字符到它们的数值的映射。 例如, Unicode 是一种16位字符集, 能够对所有已知字符进行编码, 并将其用作全球字符编码标准。

**该子**

层次结构关系的从属端。 子节点是层次结构中具有另一个节点 (更接近于根) 的节点。 另请参阅**子别名**、**父-子关系**和**父项**。

**child-alias**

引用子级的别名。 另请参阅**alias**、**子级**。

**CLSID (类标识符)**

标识 COM 组件的通用唯一标识符 (UUID)。 每个 COM 组件在 Windows 注册表中都有其 CLSID, 以便其他应用程序可以加载它。 另请参阅**ProgID**和**COM**。

**客户端层**

通常向用户提供数据并处理来自用户的输入的分布式系统的逻辑层, 有时称为*前端*。 通常情况下, 客户端层根据输入从服务器请求数据, 然后设置并显示结果。 另请参阅**中间层**、**数据源层**和**分布式应用程序**。

**COM (组件对象模型)**

一种二进制标准, 使对象能够在网络环境中进行互操作, 而不管它们是在什么语言中开发的, 或者它们驻留在哪些计算机上。 基于 COM 的技术包括 ActiveX 控件、自动化和对象链接和嵌入 (OLE)。 COM 允许对象向其他组件和主机应用程序公开其功能。 它定义对象如何公开自己以及此公开在各个进程之间以及跨网络的工作方式。 COM 还定义对象的生命周期。

**COM 组件**

二进制文件 (如 .dll、.ocx 和一些 .exe 文件), 它们支持用于提供对象的 COM 标准。 此类文件包含用于一个或多个类工厂、COM 类、注册表项机制、加载代码等的代码。

**比较运算符**

一个运算符, 用于比较两个表达式并返回一个布尔值。

一个 criteria 参数\>, 可表示为 "" (大于)、"\<(小于)、" = "(等)、"\>= "(大于或等于)、"\<= "(小于或等于)、" = "(小于或等于\<\>)、" "(不等于) 或" like "(模式匹配)。

**component**

一个对象, 它封装数据和代码, 并提供一组特定的公开提供的服务。

**复合文件**

文件的 COM 结构化存储的实现。 复合文件将单独的对象存储在单个结构化文件中, 其中包含两个主要元素: storage 对象和 stream 对象。 它们一起工作, 就像文件中的文件系统。 有关详细信息, 请参阅 Microsoft Platform SDK 中的复合文件。

在一个物理文件中绑定在一起的多个单个文件。 复合文件中的每个单独文件都可以像访问单个物理文件一样被访问。

**常量**

不更改的数值或字符串值。 命名的 ADO 枚举 (枚举常量) 可在代码中使用, 而不是实际值, 例如, **adUseClient**是一个值为3的常量。 (Const adUseClient = 3)。 另请参阅**枚举**。

**游标**

一个数据库元素, 用于控制记录导航、数据更新以及其他用户对数据库所做更改的可见性。

## <a name="d"></a>D

**数据绑定**

将应用程序的对象或控件与数据源关联的过程。 与数据源关联的控件称为*数据绑定控件*。

数据绑定控件的内容与数据库中的值相关联。 例如, 在更新**recordset**中的行时, 可以更新绑定到**recordset**对象的网格控件。 当**Recordset**检索新值时, 新值将显示在网格中。

**数据提供程序**

直接或通过服务提供程序向 ADO 应用程序公开数据的软件。 另请参阅**服务提供程序**。

**数据定形**

一种使用形式化语法 (称为 " **Shape language**") 定义专用**recordset**对象 (称为 "**整形" recordset**) 的技术, 它不仅包含数据, 还包含对其他**Recordset**对象的引用和/or 基于这些其他**Recordset**对象的计算值。

**数据源层**

表示运行 DBMS 的计算机 (如 SQL Server 数据库) 的分布式系统的逻辑层。 另请参阅**客户端层**、**中间层**、**分布式应用程序**。

**封锁**

使 COM 组件能够在网络中彼此直接通信的电线协议。 另请参阅**COM**、**组件**。

**DDL (数据定义语言)**

SQL 中的那些定义 (而不是操作数据) 的语句。 使用 DDL 创建或修改数据库的架构。 例如, **create TABLE**、 **create INDEX**、 **GRANT**和**REVOKE**是 SQL DDL 语句。

**默认流**

在使用某些 OLE db 提供程序 (如用于 Internet 发布的 Microsoft OLE db 提供程序) 时, 与**Record**或**Recordset**对象相关联的文本或二进制流 (由**stream**对象表示)。 默认流通常包含文件的内容, 如网站的根的 HTML 代码。

**分布式应用程序**

一种写入的程序, 可在网络中跨多台计算机划分处理。 通常情况下, 分布式应用程序分为演示、业务逻辑和数据存储层或*层*。 另请参阅**客户端层**、**中间层**和**数据源层**。

**断开连接的 Recordset**

客户端缓存中的**Recordset**对象, 该对象不再具有到服务器的活动连接。 如果由于某种原因 (如更新数据) 而需要再次访问原始数据源, 则必须重新建立连接。 但是, 仍然可以访问已断开连接的**Recordset**的集合、属性和方法。

**DLL (动态链接库)**

一个文件, 其中包含一个或多个编译、链接和存储与使用它们的进程分开的函数。 在进程启动时或正在运行时, 操作系统将 dll 映射到调用进程的地址空间。

**DML (数据操作语言)**

SQL 中的那些操作 (与定义数据相对) 的语句。 将选择数据库中的值, 并使用 DML 修改这些值。 例如, **INSERT**、 **UPDATE**、 **DELETE**和**SELECT**是 SQL DML 语句。

**文档源提供程序**

管理文件夹和文档的特殊类型的提供程序。 当文档由**Record**对象表示, 或文档的文件夹由**Recordset**对象表示时, 文档源提供程序将使用描述文档特征的一组唯一字段填充这些对象。而不是实际的文档本身。 另请参阅**资源记录**。

**DSN (数据源名称)**

用于将应用程序连接到特定 ODBC 数据库的信息的集合。 ODBC 驱动程序管理器使用此信息创建与数据库的连接。 DSN 可以存储在文件 (文件 DSN) 或 Windows 注册表 (计算机 DSN) 中。

**动态属性**

特定于数据提供程序或游标服务的属性。 对象的**Properties**集合将自动填充 ("动态")。 在通过特定数据提供程序连接到数据源之前, 对象没有动态属性。 另请参阅**数据提供程序**和**游标**。

**事件处理程序**

事件处理程序是在发生事件时执行的代码。 另请参阅**事件**。

**handler**

管理常见且相对简单的条件或操作 (如错误恢复或数据管理) 的例程。

**分层记录集**

**ISAPI (Internet 服务器应用程序编程接口)**

## <a name="k-m"></a>K-M

**key**

唯一标识行的表中的一个或多个列。通常用于对表编制索引。

**处理**

跨线程或进程边界对接口方法参数进行打包、发送和 unpackaging 的过程。

**中间层**

在用户界面或 web 客户端与数据库之间的分布式系统中的逻辑层。 这通常是实例化业务对象的地方。 中间层是在接收信息时生成和运行的业务规则和函数的集合。 它们通过业务规则 (可能会频繁更改) 实现此目的, 从而封装到物理上与应用程序逻辑本身分开的组件中。 也称为 "*应用程序服务器层*"。 另请参阅**分布式应用程序**、**客户端层**和**数据源层**。

**MIME (多用途 Internet 邮件扩展)**

最初开发的 Internet 协议, 允许跨异类网络、计算机和电子邮件环境交换具有丰富内容的电子邮件。 实际上, 非邮件应用程序也采用和扩展了 MIME。

MIME 是一项允许在 Internet 上发布和读取二进制数据的标准。 包含二进制数据的文件的标头包含数据的 MIME 类型;这会通知客户端程序 (例如, web 浏览器和邮件包) 需要以不同的方式处理数据, 而不是处理直线文本。 例如, 包含 jpeg 图形的 web 文档的标题包含特定于 jpeg 文件格式的 MIME 类型。 这将允许浏览器显示文件及其 JPEG 查看器 (如果存在)。

**ODBC（开放式数据库连接）**

用于连接到各种数据源的标准编程语言接口。 这通常通过 "控制面板" 访问, 其中可分配数据源名称 (dsn) 以使用特定的 ODBC 驱动程序。

**OLE DB**

使用 COM 公开各种源中的数据的一组接口。 OLE DB 接口为应用程序提供了对各种信息源中存储的数据的统一访问。 这些接口支持与数据源适合的 DBMS 功能的数量, 使其可以共享其数据。 另请参阅**COM**。

**乐观锁定**

一种锁定类型, 其中包含一个或多个记录 (包括正在编辑的记录) 的数据页面, 只有在使用**update**方法更新记录时, 其他用户才能使用该锁定, 但在调用**更新**之前和之后都可以使用它们。.

**序号值**

## <a name="p"></a>P

**参数化命令**

允许您在执行命令前设置参数值的查询或命令。 例如, 可以通过在 sql 字符串中嵌入参数标记 (由 '？ ' 字符指定) 来参数化 sql 字符串。 然后, 应用程序为每个参数指定值并执行命令。

**母语**

层次结构关系的控制侧。 在层次结构中, 父节点在层次结构中直接在其下方包含一个或多个子节点。 另请参阅**父别名**、**父-子关系**和**子级**。

**parent-alias**

引用父项的别名。 另请参阅**别名**和**父**。

**父子关系**

层次结构中的一个关系, 其中的父级的级别较高, 与一个或多个子项直接关联。 子级的级别低于一个级别, 并且必须具有一个父项。 另请参阅**parent**、 **child**。

**下来**

将数据保存为永久性状态, 如将**Recordset**保存到文件中。

**保守式锁定**

一种锁定类型, 其中包含一个或多个记录 (包括正在编辑的记录) 的页面对其他用户不可用, 以确保将进行更新。 保守式锁定行为由 OLE DB 提供程序定义。 通常情况下, 记录在编辑时处于锁定状态, 并且在**更新**方法完成前仍不可用。

当使用**LockType**参数或属性设置为**adLockPessimistic**时, 打开**Recordset**对象时, 将启用保守式锁定。 另请参阅**开放式锁定**。

**处理**

基于预分配资源的集合 (如对象或数据库连接) 的性能优化。 从池 (而不是创建新资源) 中绘制现有资源效率更高。

**ProgID (编程标识符)**

一个由 COM 应用程序映射到 Windows 注册表的唯一名称。 ADO 连接的 ProgID 是 "ADODB"。Connection "。 另请参阅**CLSID**、 **COM**。

**代理**

特定于接口的对象, 提供客户端调用在不同的执行环境中运行的应用程序对象 (如在不同的线程上或在另一个进程中) 时所需的参数封送和通信。 代理与客户端进行通信, 并与与被调用的 application 对象位于一起的相应存根进行通信。 另请参阅**存根**。

## <a name="r"></a>R

**相对 URL**

指定 Internet 或 intranet 上的资源的部分限定的 URL, 其位置相对于绝对 URL 或等效 ADO Connection 对象指定的起始点。 实际上, 串联的绝对 url 和相对 url consitute 完整的 url。 另请参阅**url**和**绝对 URL**。

**远程数据源**

在另一台计算机上, 而不是在本地系统 (客户端应用程序运行的位置) 上存在的数据源。

**资源记录**

文档源提供程序中的一条记录, 其中包含文件夹或文档的定义和说明的字段。 文档本身并不包含在资源记录中, 但通常可以通过默认流或包含 URL 的资源记录中的字段进行访问。 另请参阅**文档源提供程序**、**默认流**和**URL**。

**root**

层次结构树结构中的顶层。 根节点没有父节点, 但可能有子项。 另请参阅**层次结构**、**树**、**父级**和**子级**。

**rowset**

数据源中的一组行, 所有这些行都具有相同的字段架构。 一个行集可以表示表中的所有字段或部分字段。 行集还可以表示由查询或两个或更多表的联接创建的虚拟表。 在 ADO 中, 行集由**Recordset**对象表示。

## <a name="s"></a>S

**架构**

数据库管理系统 (DBMS) 的数据库说明, 通常使用 DBMS 提供的数据定义语言生成。 架构定义数据库的属性, 如表、列和属性。

**scope**

对象或变量的引用范围或视图或表中的记录范围。 例如, 只能在定义局部变量的过程中引用这些变量。 公共变量可从应用程序中的任何位置访问。 如果对象位于定义的搜索路径中, 则这些对象 (如当前数据库) 在范围内。 在许多命令中, 可以使用 Scope 子句指定记录范围。

**服务提供程序**

通过生成和使用数据以及在 ADO 应用程序中补充功能来封装服务的软件。 它是一个不直接公开数据的提供程序, 而是提供一种服务, 例如查询处理。 服务提供程序可能会处理由数据提供程序提供的数据。 另请参阅**data provider**。

**整形 Recordset**

一种**Recordset** , 其列已明确定义为仅包含数据, 还包含对其他**recordset**对象和/或基于其他**recordset**对象的计算值的引用 (称为 "章节")。

**同辈**

层次结构中相同级别上的任意两个或更多个节点。 层次结构中的根节点没有同级。

**存储过程**

代码的预编译集合, 如 SQL 语句和可选的控制流语句, 它们存储在一个名称下并作为一个单元处理。 存储过程存储在数据库中;可以使用来自应用程序的一个调用来执行这些操作, 并允许用户声明的变量、条件执行和其他功能强大的编程功能。

**根**

特定于接口的对象, 提供应用程序对象从运行在不同的执行环境中的客户端 (例如, 在不同的线程上或在另一个进程中) 中接收调用所需的参数封送和通信。 存根与 application 对象放置在一起, 并与调用它的客户端上的相应代理进行通信。 另请参阅**proxy**。

请参阅**子级**。

**同步操作**

由代码启动的操作, 在下一操作可能开始之前完成。 另请参阅**异步操作**。

## <a name="t-w"></a>T W

**树**

一个表示元素 (节点) 之间的层次结构关系的结构。 树的顶层有一个节点 (根)。 在根下, 可以有多个子级。 每个子项又可能是其他子级的父项, 因此像树这样的分支。 包含文档和其他文件夹的文件夹是树状结构的典型示例。 另请参阅**层次结构**、**节点**、**根**、**子**和**父**。

**URL (统一资源定位器)**

指定驻留在 Internet 或 intranet 上的资源的位置。 完整的 URL 包含一个方案 (如 FTP、HTTP、mailto、文件等), 后跟冒号、服务器名称和资源的完整路径 (如文档、图形或其他文件)。 url 的一些示例如下:  
  
- https://www.domain.com/default.html  
- ftp://ftp.server.somewhere/ftp.file  
  
- ftp://ftp.server.somewhere/ftp.file  
- file://Server/Share/File.doc

另请参阅**绝对 url**和**相对 url**。

**web 服务器**

向 intranet 和 Internet 用户提供 web 服务和页面的计算机。
