# <a name="supporting-event-notification"></a><span data-ttu-id="2ee73-103">支持事件通知</span><span class="sxs-lookup"><span data-stu-id="2ee73-103">Supporting Event Notification</span></span>

  
  
<span data-ttu-id="2ee73-104">**适用于**：Outlook 2013 | Outlook 2016</span><span class="sxs-lookup"><span data-stu-id="2ee73-104">**Applies to**: Outlook 2013 | Outlook 2016</span></span> 
  
<span data-ttu-id="2ee73-105">由于支持事件通知可能很复杂, 因此 MAPI 提供了三个支持对象方法来实现过程中最困难的部分。</span><span class="sxs-lookup"><span data-stu-id="2ee73-105">Because supporting event notification can be complicated, MAPI supplies three support object methods that implement the most difficult parts of the process.</span></span> <span data-ttu-id="2ee73-106">这些方法可用作一个单元, 提供程序必须使用全部三个或全部三个。</span><span class="sxs-lookup"><span data-stu-id="2ee73-106">These methods work as a unit, and a provider must use all three or none of them.</span></span>
  
<span data-ttu-id="2ee73-107">MAPI 支持方法使用通知密钥来管理通知接收器与生成通知的对象之间的连接。</span><span class="sxs-lookup"><span data-stu-id="2ee73-107">The MAPI support methods use notification keys to manage the connections between the advise sinks and the objects that generate the notifications.</span></span> <span data-ttu-id="2ee73-108">通知密钥是一个[NOTIFKEY](notifkey.md)结构, 其中包含用于在各个进程之间标识对象的二进制数据。</span><span class="sxs-lookup"><span data-stu-id="2ee73-108">A notification key is a [NOTIFKEY](notifkey.md) structure that contains binary data that identifies an object across processes.</span></span> <span data-ttu-id="2ee73-109">通知密钥通常是从通知源对象的长期条目标识符中复制的。</span><span class="sxs-lookup"><span data-stu-id="2ee73-109">A notification key is typically copied from the long-term entry identifier of the advise source object.</span></span> <span data-ttu-id="2ee73-110">如果客户端在调用中提供了一个条目标识符, \*\*\*\* 则可以将其用于通知密钥。</span><span class="sxs-lookup"><span data-stu-id="2ee73-110">If the client has supplied an entry identifier in the call to **Advise**, you can use it for the notification key.</span></span> <span data-ttu-id="2ee73-111">如果**建议**的_lpEntryID_参数为 NULL, 则使用最外面的容器对象 (如邮件存储) 的条目标识符。</span><span class="sxs-lookup"><span data-stu-id="2ee73-111">If the  _lpEntryID_ parameter to **Advise** is NULL, use the entry identifier of the outermost possible container object, such as the message store.</span></span> 
  
<span data-ttu-id="2ee73-112">若要使用支持方法, 请在客户端呼叫您的**Advise**方法以注册通知时, 调用[IMAPISupport:: 订阅](imapisupport-subscribe.md)。</span><span class="sxs-lookup"><span data-stu-id="2ee73-112">To use the support methods, call [IMAPISupport::Subscribe](imapisupport-subscribe.md) whenever a client calls your **Advise** method to register for a notification.</span></span> <span data-ttu-id="2ee73-113">为您的通知源对象分配一个[NOTIFKEY](notifkey.md)结构并创建一个唯一的通知密钥。</span><span class="sxs-lookup"><span data-stu-id="2ee73-113">Allocate a [NOTIFKEY](notifkey.md) structure and create a unique notification key for your advise source object.</span></span> <span data-ttu-id="2ee73-114">例如, 接收到特定文件夹中的邮件时, 系统会提示通知客户端的邮件存储提供程序会为该文件夹创建一个通知密钥。</span><span class="sxs-lookup"><span data-stu-id="2ee73-114">For example, a message store provider that is prompted to notify a client when a message is received into a particular folder creates a notification key for that folder.</span></span> <span data-ttu-id="2ee73-115">在要**订阅**的调用中传递指向**NOTIFKEY**结构的指针, 并将指针指向客户端的通知接收器。</span><span class="sxs-lookup"><span data-stu-id="2ee73-115">Pass a pointer to the **NOTIFKEY** structure in the call to **Subscribe** along with a pointer to the client's advise sink.</span></span> <span data-ttu-id="2ee73-116">**订阅**呼叫通知接收器的[IUnknown:: AddRef](https://msdn.microsoft.com/library/b4316efd-73d4-4995-b898-8025a316ba63%28Office.15%29.aspx)方法增加其引用计数和 MAPI 将保留指针, 直到取消注册。</span><span class="sxs-lookup"><span data-stu-id="2ee73-116">**Subscribe** calls the advise sink's [IUnknown::AddRef](https://msdn.microsoft.com/library/b4316efd-73d4-4995-b898-8025a316ba63%28Office.15%29.aspx) method to increment its reference count and MAPI retains the pointer until the registration is canceled.</span></span> 
  
<span data-ttu-id="2ee73-117">您可以传递 NOTIFY_SYNC 标志来**订阅\*\*\*\*通知**同步运行的请求, 而不会返回, 直到对已注册的通知接收器的[IMAPIAdviseSink:: OnNotify](imapiadvisesink-onnotify.md)方法进行了所有调用。</span><span class="sxs-lookup"><span data-stu-id="2ee73-117">You can pass the NOTIFY_SYNC flag to **Subscribe** to request that **Notify** behave synchronously and not return until it has made all calls to the [IMAPIAdviseSink::OnNotify](imapiadvisesink-onnotify.md) methods of registered advise sinks.</span></span> <span data-ttu-id="2ee73-118">仅为自己的内部使用设置此标志。</span><span class="sxs-lookup"><span data-stu-id="2ee73-118">Set this flag only for your own internal use.</span></span> <span data-ttu-id="2ee73-119">当您响应客户端**通知**调用时, 请不要设置它。</span><span class="sxs-lookup"><span data-stu-id="2ee73-119">Do not set it when you respond to a client **Advise** call.</span></span> <span data-ttu-id="2ee73-120">客户端和提供程序之间的事件通知始终是异步的。</span><span class="sxs-lookup"><span data-stu-id="2ee73-120">Event notification between clients and providers is always asynchronous.</span></span> <span data-ttu-id="2ee73-121">也就是说, MAPI 可确保发生事件时的调用在发出任何**OnNotify**调用之前返回到客户端。</span><span class="sxs-lookup"><span data-stu-id="2ee73-121">That is, MAPI guarantees that the call during which an event happens will return to the client before any of the **OnNotify** calls are made.</span></span> 
  
<span data-ttu-id="2ee73-122">如果设置了 NOTIFY_SYNC 标志, 请勿对任何建议接收器对象进行任何更改, 并且不会传递由[HrThisThreadAdviseSink](hrthisthreadadvisesink.md)创建的包装建议接收器来进行**订阅**。</span><span class="sxs-lookup"><span data-stu-id="2ee73-122">If you set the NOTIFY_SYNC flag, do not make any changes to any of the advise sink objects, and do not pass a wrapper advise sink created by [HrThisThreadAdviseSink](hrthisthreadadvisesink.md) to **Subscribe**.</span></span> <span data-ttu-id="2ee73-123">**HrThisThreadAdviseSink**创建建议接收器的线程安全版本, 仅用于异步通知。</span><span class="sxs-lookup"><span data-stu-id="2ee73-123">**HrThisThreadAdviseSink** creates a thread-safe version of an advise sink to be used with asynchronous notification only.</span></span> 
  
<span data-ttu-id="2ee73-124">如果为同步通知注册的通知接收器从设置了 CALLBACK_DISCONTINUE 标志的**OnNotify**返回, 则[IMAPISupport:: Notify](imapisupport-notify.md)将设置 NOTIFY_CANCELED 标志并返回, 而不会对**OnNotify**进行任何调用。</span><span class="sxs-lookup"><span data-stu-id="2ee73-124">If an advise sink registered for synchronous notification returns from **OnNotify** with the CALLBACK_DISCONTINUE flag set, [IMAPISupport::Notify](imapisupport-notify.md) sets the NOTIFY_CANCELED flag and returns without making any calls to **OnNotify**.</span></span> 
  
<span data-ttu-id="2ee73-125">**订阅**返回后, 您将不再需要在客户端的通知接收器副本上保留。</span><span class="sxs-lookup"><span data-stu-id="2ee73-125">Once **Subscribe** has returned, you will no longer have any need to hold onto your copy of the client's advise sink.</span></span> <span data-ttu-id="2ee73-126">调用其[IUnknown:: release](https://msdn.microsoft.com/library/4b494c6f-f0ee-4c35-ae45-ed956f40dc7a%28Office.15%29.aspx)方法将其释放。</span><span class="sxs-lookup"><span data-stu-id="2ee73-126">Call its [IUnknown::Release](https://msdn.microsoft.com/library/4b494c6f-f0ee-4c35-ae45-ed956f40dc7a%28Office.15%29.aspx) method to release it.</span></span> <span data-ttu-id="2ee73-127">**订阅**返回应返回到客户端的非零连接号码。</span><span class="sxs-lookup"><span data-stu-id="2ee73-127">**Subscribe** returns a nonzero connection number that you should return to the client.</span></span> <span data-ttu-id="2ee73-128">连接号码代表通知源和通知接收器之间的链接。</span><span class="sxs-lookup"><span data-stu-id="2ee73-128">The connection number represents the link between the advise source and the advise sink.</span></span> <span data-ttu-id="2ee73-129">在客户端成功调用**Unadvise**之前, 它将一直保持有效。</span><span class="sxs-lookup"><span data-stu-id="2ee73-129">It remains valid until the client makes a successful call to **Unadvise**.</span></span> 
  
<span data-ttu-id="2ee73-130">当客户端准备好取消注册时, 它会调用您的**Unadvise**方法。</span><span class="sxs-lookup"><span data-stu-id="2ee73-130">When the client is ready to cancel a registration, it calls your **Unadvise** method.</span></span> <span data-ttu-id="2ee73-131">将连接号码从**Unadvise**调用传递到[IMAPISupport:: 退订](imapisupport-unsubscribe.md)。</span><span class="sxs-lookup"><span data-stu-id="2ee73-131">Pass the connection number from the **Unadvise** call to [IMAPISupport::Unsubscribe](imapisupport-unsubscribe.md).</span></span> <span data-ttu-id="2ee73-132">**取消订阅**将呼叫通知接收器的**IUnknown:: Release**方法。</span><span class="sxs-lookup"><span data-stu-id="2ee73-132">**Unsubscribe** calls the advise sink's **IUnknown::Release** method.</span></span> <span data-ttu-id="2ee73-133">与**建议**和**Unadvise**一样,**订阅**和**取消订阅**的呼叫必须配对。</span><span class="sxs-lookup"><span data-stu-id="2ee73-133">As with **Advise** and **Unadvise**, calls to **Subscribe** and **Unsubscribe** must be paired.</span></span> <span data-ttu-id="2ee73-134">您必须对要**订阅**的每个呼叫发出一个**取消订阅**的调用。</span><span class="sxs-lookup"><span data-stu-id="2ee73-134">You must make one call to **Unsubscribe** for every call that is made to **Subscribe**.</span></span> <span data-ttu-id="2ee73-135">但是, 每次调用您的**Advise**方法时, 不必调用**订阅**。</span><span class="sxs-lookup"><span data-stu-id="2ee73-135">However, you do not have to call **Subscribe** every time your **Advise** method is called.</span></span> <span data-ttu-id="2ee73-136">相反, 您可以调用它来设置内部通知。</span><span class="sxs-lookup"><span data-stu-id="2ee73-136">Conversely, you can call it for setting up internal notifications.</span></span> 
  
<span data-ttu-id="2ee73-137">在事件发生时, 分配适用于事件的类型的一个或多个[通知](notification.md)结构, 并调用[IMAPISupport:: Notify](imapisupport-notify.md)。</span><span class="sxs-lookup"><span data-stu-id="2ee73-137">When an event occurs, allocate one or more [NOTIFICATION](notification.md) structures of the type appropriate for the event and call [IMAPISupport::Notify](imapisupport-notify.md).</span></span> <span data-ttu-id="2ee73-138">**通知**为每个已注册的建议接收器生成一个通知。</span><span class="sxs-lookup"><span data-stu-id="2ee73-138">**Notify** generates a notification for each registered advise sink.</span></span> <span data-ttu-id="2ee73-139">应将[通知](notification.md)结构的所有未使用成员设置为零。</span><span class="sxs-lookup"><span data-stu-id="2ee73-139">You should set all the unused members of the [NOTIFICATION](notification.md) structure to zero.</span></span> <span data-ttu-id="2ee73-140">这种初始化**通知**结构的技术可以帮助客户端创建更小、更快速且不易出错的**OnNotify**实现。</span><span class="sxs-lookup"><span data-stu-id="2ee73-140">This technique for initializing the **NOTIFICATION** structure can help clients create smaller, faster, and less error-prone **OnNotify** implementations.</span></span> 
  
<span data-ttu-id="2ee73-141">请注意, 每个事件都需要单独的**通知**结构, 即使是同一类型的多个事件也是如此。</span><span class="sxs-lookup"><span data-stu-id="2ee73-141">Note that a separate **NOTIFICATION** structure is necessary for each event, even for multiple events of the same type.</span></span> <span data-ttu-id="2ee73-142">例如, 如果为特定表上的表通知注册了三个客户端, 并且向表中添加了五行, 则必须为**通知**调用创建五个**OBJECT_NOTIFICATION**结构。</span><span class="sxs-lookup"><span data-stu-id="2ee73-142">For example, if three clients are registered for table notification on a particular table and five rows are added to the table, you must create five **OBJECT_NOTIFICATION** structures for your **Notify** call.</span></span> <span data-ttu-id="2ee73-143">这样的批处理通知导致性能优于呼叫**通知**五次。</span><span class="sxs-lookup"><span data-stu-id="2ee73-143">A batch notification such as this results in better performance than calling **Notify** five times.</span></span> <span data-ttu-id="2ee73-144">对于每个**通知**呼叫, MAPI 都会调用每个已注册的建议接收器的[IMAPIAdviseSink:: OnNotify](imapiadvisesink-onnotify.md)方法。</span><span class="sxs-lookup"><span data-stu-id="2ee73-144">For each **Notify** call, MAPI calls the [IMAPIAdviseSink::OnNotify](imapiadvisesink-onnotify.md) method of every registered advise sink.</span></span> <span data-ttu-id="2ee73-145">如果没有已注册的建议接收器, MAPI 将忽略该呼叫。</span><span class="sxs-lookup"><span data-stu-id="2ee73-145">If there are no registered advise sinks, MAPI ignores the call.</span></span>
