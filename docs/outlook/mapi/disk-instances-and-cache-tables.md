# <a name="disk-instances-and-cache-tables"></a>磁盘实例和缓存表

**适用于**：Outlook 2013 | Outlook 2016 
  
若要激活窗体, 其可执行文件必须在用户的计算机上可用。 如果它们不可用, 则必须将它们从表单库复制到本地磁盘。 若要执行此操作, 默认表单管理器会在用户的 Windows 目录中创建一个子目录, 以包含该表单的可执行文件 (。exe。HLPs)。 此目录称为表单的磁盘实例。
  
默认表单管理器维护所有磁盘实例的表, 以便在已存在磁盘实例时可以使用该实例, 而无需将文件从表单库复制到用户磁盘。 磁盘实例表是作为最常用的缓存进行管理。 如果需要新的磁盘实例, 则会将其复制到用户的计算机上, 以替换最常用的磁盘实例。 然后更新磁盘实例缓存表以反映最新的配置。 磁盘缓存的大小是一个用户可配置的选项, 使用户能够以更高的速度平衡可用磁盘容量。
  
除了磁盘实例缓存之外, 默认表单管理器还维护一个正在运行的实例表, 该表列出了用户计算机上所有正在运行的表单服务器实例。 这将使用 MAPI 将空闲表单实例保持在不可见状态中运行, 直到窗体服务器的邮件类的窗体被激活。 换句话说, 表单服务器可以缓存在内存中, 以最大限度地减少表单可执行文件在表单库中的的次数, 并从磁盘或网络中加载到内存中。 与磁盘实例缓存一样, 正在运行的实例缓存的行为方式最少, 因此可以从缓存中清除正在运行的窗体实例, 以便为另一个窗体实例留出空间。 在表单库在表单库中搜索表单库之前, 将在此缓存中搜索运行的表单服务器实例。
  
> [!NOTE]
> 默认表单管理器在用户的工作站上安装表单时显示一个进度指示器, 使用户可以取消该操作。 如果用户与表单服务器的可执行文件的连接是在低带宽网络上, 则此方法尤其有用。
