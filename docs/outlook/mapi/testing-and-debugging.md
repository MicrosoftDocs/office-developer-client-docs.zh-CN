# <a name="testing-and-debugging"></a>测试和调试

  
  
**适用于**：Outlook 2013 | Outlook 2016 
  
测试策略因开发客户端还是服务提供商而异。 由于客户端应用程序需要一个或多个服务提供程序来运行，因此应在具有不同服务提供程序集的环境中测试客户端。
  
但是，在与其他提供程序集成之前，应单独测试服务提供商。 MAPI 提供了用于测试特定类型服务提供商的功能的工具。 [MFCMAPI](https://go.microsoft.com/fwlink/?LinkId=124154)示例应用程序演示如何测试通讯簿提供程序的功能以及如何使用邮件存储提供程序。
