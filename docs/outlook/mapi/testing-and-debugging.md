# <a name="testing-and-debugging"></a>测试和调试

  
  
**适用于**： Outlook 
  
测试策略不同，具体取决于是否正在开发的客户端或服务提供程序。 因为客户端应用程序需要一个或多个服务提供商进行操作，则应具有不同的服务提供程序集的环境中测试客户端。
  
但是，应在之前要与其他提供程序集成的隔离中测试服务提供商。 MAPI 提供了为了测试特定类型的服务提供程序的功能的工具。 [MFCMAPI](http://go.microsoft.com/fwlink/?LinkId=124154)示例应用程序演示如何测试通讯簿提供程序的功能和适用于一条消息存储提供程序。
