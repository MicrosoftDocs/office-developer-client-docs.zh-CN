<span data-ttu-id="3b302-113">建议接收器对象是最常见的类型。</span><span class="sxs-lookup"><span data-stu-id="3b302-113">Advise sink objects are the most common type.</span></span> <span data-ttu-id="3b302-114">建议接收器通常由客户端应用程序实现, 以接收通讯簿和邮件存储通知, 并支持[IMAPIAdviseSink: IUnknown](imapiadvisesinkiunknown.md)接口。</span><span class="sxs-lookup"><span data-stu-id="3b302-114">Advise sinks are typically implemented by client applications to receive address book and message store notifications and support the [IMAPIAdviseSink : IUnknown](imapiadvisesinkiunknown.md) interface.</span></span> <span data-ttu-id="3b302-115">**IMAPIAdviseSink**包含一个方法, [IMAPIAdviseSink:: OnNotify](imapiadvisesink-onnotify.md)。</span><span class="sxs-lookup"><span data-stu-id="3b302-115">**IMAPIAdviseSink** contains a single method, [IMAPIAdviseSink::OnNotify](imapiadvisesink-onnotify.md).</span></span> <span data-ttu-id="3b302-116">窗体和视图的建议接收器不常见;它们的实现是为了接收有关自定义窗体更改的通知。</span><span class="sxs-lookup"><span data-stu-id="3b302-116">Form and view advise sinks are less common; <span data-ttu-id="3b302-120">通知源对象由服务提供商和 MAPI 实现。</span><span class="sxs-lookup"><span data-stu-id="3b302-120">Advise source objects are implemented by service providers and by MAPI.</span></span> <span data-ttu-id="3b302-121">并非所有服务提供程序都支持事件通知;它是可选的, 但强烈建议这样做。</span><span class="sxs-lookup"><span data-stu-id="3b302-121">Not all service providers support event notification; it is optional, but strongly recommended.</span></span> <span data-ttu-id="3b302-122">邮件存储和通讯簿提供程序通常支持其内容和层次结构表上的多个对象和表通知的对象通知。</span><span class="sxs-lookup"><span data-stu-id="3b302-122">Message store and address book providers usually support object notifications on several of their objects and table notifications on their contents and hierarchy tables.</span></span> <span data-ttu-id="3b302-123">传输提供程序不直接支持通知;它们依赖于与客户端进行通信的替代方法。</span><span class="sxs-lookup"><span data-stu-id="3b302-123">Transport providers do not support notifications directly; they rely on alternative methods of communication with clients.</span></span>
  
<span data-ttu-id="3b302-124">与建议接收器不同, 建议源对象不是 MAPI 对象的唯一类型。</span><span class="sxs-lookup"><span data-stu-id="3b302-124">Unlike advise sinks, advise source objects are not a unique type of MAPI object.</span></span> <span data-ttu-id="3b302-125">许多 MAPI 对象 (如邮件存储和表) 可以承担建议源的角色。</span><span class="sxs-lookup"><span data-stu-id="3b302-125">Many MAPI objects, such as message stores and tables, can take on the role of advise source.</span></span> <span data-ttu-id="3b302-126">"建议源" 是执行以下操作的任何 MAPI 对象:</span><span class="sxs-lookup"><span data-stu-id="3b302-126">An advise source is any MAPI object that does the following:</span></span>
  
- <span data-ttu-id="3b302-127">实现用于接收通知注册的**建议**方法。</span><span class="sxs-lookup"><span data-stu-id="3b302-127">Implements an **Advise** method to receive notification registrations.</span></span> 
    
- <span data-ttu-id="3b302-128">实现**Unadvise**方法以接收通知取消。</span><span class="sxs-lookup"><span data-stu-id="3b302-128">Implements an **Unadvise** method to receive notification cancellations.</span></span> 
    
- <span data-ttu-id="3b302-129">通过调用**IMAPIAdviseSink:: OnNotify**方法, 向已注册的相应通知接收器对象生成适当类型的通知。</span><span class="sxs-lookup"><span data-stu-id="3b302-129">Generates notifications of the appropriate type to the appropriate advise sink objects that have registered by calling their **IMAPIAdviseSink::OnNotify** methods.</span></span> 
    
<span data-ttu-id="3b302-130">实现通知接收器对象的客户端在想要注册通知时调用**建议**, 在大多数情况下, 在应进行注册的对象的条目标识符中传递, 并在需要时使用**Unadvise**来取消注册.</span><span class="sxs-lookup"><span data-stu-id="3b302-130">Clients that implement advise sink objects call **Advise** when they want to register for a notification, in most cases passing in the entry identifier of the object with which registration should occur, and **Unadvise** when they want to cancel the registration.</span></span> <span data-ttu-id="3b302-131">客户端将一个参数传递给 "**建议**", 指示他们要监视的几种类型的事件中的哪一种。</span><span class="sxs-lookup"><span data-stu-id="3b302-131">Clients pass a parameter to **Advise** that indicates which of the several types of events they want to monitor.</span></span> <span data-ttu-id="3b302-132">**建议**返回一个非零的数字, 表示通知接收器和通知源之间的成功连接。</span><span class="sxs-lookup"><span data-stu-id="3b302-132">**Advise** returns a nonzero number that represents a successful connection between the advise sink and advise source.</span></span>

<span data-ttu-id="3b302-138">服务提供程序可以手动实现对通知的支持, 也可以利用三个**IMAPISupport**方法提供的帮助: [IMAPISupport:: 订阅](imapisupport-subscribe.md)、 [IMAPISupport:: 退订](imapisupport-unsubscribe.md)和[IMAPISupport:: Notify](imapisupport-notify.md).</span><span class="sxs-lookup"><span data-stu-id="3b302-138">Service providers can implement support for notifications manually or take advantage of the help provided in three **IMAPISupport** methods: [IMAPISupport::Subscribe](imapisupport-subscribe.md), [IMAPISupport::Unsubscribe](imapisupport-unsubscribe.md), and [IMAPISupport::Notify](imapisupport-notify.md).</span></span> <span data-ttu-id="3b302-139">**订阅**和**取消订阅**方法处理提供程序的通知注册和注销;**Notify**方法处理在适当的时候发送通知。</span><span class="sxs-lookup"><span data-stu-id="3b302-139">The **Subscribe** and **Unsubscribe** methods handle notification registration and deregistration for providers; the **Notify** method handles sending notifications when appropriate.</span></span> 
  
<span data-ttu-id="3b302-140">若要将支持对象方法用于通知注册, 服务提供程序将调用[IMAPISupport:: 订阅](imapisupport-subscribe.md)其**建议**方法, 并\*\*\*\* 传递给客户端传递给**建议**的建议接收器指针。</span><span class="sxs-lookup"><span data-stu-id="3b302-140">To use the support object methods for notification registration, service providers call [IMAPISupport::Subscribe](imapisupport-subscribe.md) in their **Advise** methods and pass to **Subscribe** the advise sink pointer that clients pass to **Advise**.</span></span> <span data-ttu-id="3b302-141">如果条目标识符作为输入参数传递, 以指定建议源, 则服务提供程序会将其转换为二进制密钥。</span><span class="sxs-lookup"><span data-stu-id="3b302-141">If an entry identifier is passed as an input parameter to specify an advise source, service providers convert it to a binary key.</span></span> <span data-ttu-id="3b302-142">**订阅**创建唯一的连接号码, 该号码是服务提供商返回给客户端的号码。</span><span class="sxs-lookup"><span data-stu-id="3b302-142">**Subscribe** creates a unique connection number and it is this number that service providers return to clients.</span></span> <span data-ttu-id="3b302-143">在**通知**调用完成后, 服务提供商可以随时释放客户端的建议接收器对象指针。</span><span class="sxs-lookup"><span data-stu-id="3b302-143">Service providers can release the client's advise sink object pointer at any time after the **Advise** call has completed.</span></span>
